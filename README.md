## 1. 为什么需要去除异常值？

### 1.1 异常值对统计分析的影响

在进行数据分析时，**异常值 (Outliers)** 会严重扭曲统计结果：

```
问题：
┌─────────────────────────────────────────────────────────────────┐
│  原始数据示例: [0.40, 0.42, 0.45, 0.38, 0.44, 0.20, 0.73]      │
│                                              ↑       ↑         │
│                                           异常小   异常大      │
├─────────────────────────────────────────────────────────────────┤
│  影响:                                                         │
│  • 均值被拉向异常值方向                                         │
│  • 标准差被人为放大                                             │
│  • 正态分布拟合失真                                             │
│  • Baseline 阈值不准确                                         │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 我们数据中的异常值

在本次分析中，我们发现了以下异常值：

**环境泛化性数据 (68个样本):**
- 异常小值: 0.20, 0.26 (远低于正常范围)
- 异常大值: 0.64, 0.73 (远高于正常范围)
- 去除后剩余: 64个样本

**夹爪熵数据 (70个样本):**
- 异常小值: 3.68 (远低于正常范围)
- 去除后剩余: 69个样本

### 1.3 去除异常值的必要性

```
去除前 vs 去除后对比:

环境泛化性:
┌──────────────┬──────────────┬──────────────┐
│              │    去除前    │    去除后    │
├──────────────┼──────────────┼──────────────┤
│ 样本数       │     68       │     64       │
│ 均值 (μ)     │   0.4486     │   0.4479     │
│ 标准差 (σ)   │   0.0870     │   0.0684     │  ← 降低21%
│ 范围         │ [0.20, 0.73] │ [0.28, 0.59] │
└──────────────┴──────────────┴──────────────┘

夹爪熵:
┌──────────────┬──────────────┬──────────────┐
│              │    去除前    │    去除后    │
├──────────────┼──────────────┼──────────────┤
│ 样本数       │     70       │     69       │
│ 均值 (μ)     │   6.1476     │   6.1834     │
│ 标准差 (σ)   │   0.6715     │   0.6066     │  ← 降低10%
│ 范围         │ [3.68, 7.12] │ [4.81, 7.12] │
└──────────────┴──────────────┴──────────────┘
```

**结论**: 去除异常值后，标准差更能反映数据的真实变异程度，使得基于 σ 的阈值设定更加合理。

## 2. IQR 方法详解

### 2.1 什么是 IQR？

**IQR (Interquartile Range，四分位距)** 是一种稳健的统计量，用于衡量数据的离散程度。

```
数据排序后的分位数定义:
                                    
    最小值   Q1(25%)   Q2(中位数)   Q3(75%)   最大值
      ↓        ↓          ↓           ↓         ↓
   ───●────────●──────────●───────────●─────────●───
      │        │◄─────────────────────►│         │
      │        │         IQR          │         │
      │        │    (四分位距)         │         │
```

**计算公式:**
```
IQR = Q3 - Q1 = 第75百分位数 - 第25百分位数
```

### 2.2 IQR 方法如何检测异常值？

IQR 方法使用以下规则判定异常值：

```
异常值判定规则 (k = 1.5，标准设定):

    下界 = Q1 - 1.5 × IQR
    上界 = Q3 + 1.5 × IQR
    
    如果 数据点 < 下界  →  异常小值
    如果 数据点 > 上界  →  异常大值
```

**可视化:**
```
                   正常范围
         ◄─────────────────────────────►
         │                             │
    ─────┼──────●────────●────────●────┼─────
         │      Q1       Q2       Q3   │
    下界 ─┘      │◄──── IQR ────►│     └─ 上界
    (Q1-1.5×IQR)                    (Q3+1.5×IQR)
    
    ×    ×                              ×    ×
   异常值                              异常值
```

### 2.3 我们数据中的 IQR 计算

**环境泛化性:**
```
Q1 (25%) = 0.400
Q3 (75%) = 0.490
IQR = 0.490 - 0.400 = 0.090

下界 = 0.400 - 1.5 × 0.090 = 0.265
上界 = 0.490 + 1.5 × 0.090 = 0.625

检测到的异常值: 0.20, 0.26 (< 0.265), 0.64, 0.73 (> 0.625)
```

**夹爪熵:**
```
Q1 (25%) = 5.693
Q3 (75%) = 6.727
IQR = 6.727 - 5.693 = 1.034

下界 = 5.693 - 1.5 × 1.034 = 4.142
上界 = 6.727 + 1.5 × 1.034 = 8.278

检测到的异常值: 3.68 (< 4.142)
```

### 2.4 为什么选择 IQR 而不是其他方法？

| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **IQR** | 对异常值稳健，不受极端值影响 | 对小样本可能过于保守 | ✅ 中等样本量，可能有极端值 |
| Z-Score | 简单直观 | 均值和标准差本身受异常值影响 | 已知正态分布 |
| MAD | 非常稳健 | 计算复杂 | 严重偏态分布 |

**我们选择 IQR 的理由:**
1. 样本量适中 (68-70个)
2. 数据可能存在采集异常（如设备故障、操作失误）
3. IQR 方法在统计学界被广泛接受

## 3. CDF 详解

### 3.1 什么是 CDF？

**CDF (Cumulative Distribution Function，累积分布函数)** 描述随机变量小于或等于某个值的概率。

**数学定义:**
```
F(x) = P(X ≤ x) = 概率（随机变量 X 小于等于 x）
```

**直观理解:**
```
CDF 曲线示意图:

概率
1.0 ┤                              ●────────
    │                           ●
0.8 ┤                        ●
    │                     ●
0.6 ┤                  ●
    │               ●
0.4 ┤            ●
    │         ●
0.2 ┤      ●
    │   ●
0.0 ┼●─────────────────────────────────────
    └───────────────────────────────────────→ x
         小值                          大值
         
解读: CDF(x) = 0.3 意味着 30% 的数据点 ≤ x
```

### 3.2 CDF 的关键性质

```
CDF 的性质:
┌────────────────────────────────────────────────────────────────┐
│ 1. 单调递增: 值越大，CDF越高                                    │
│ 2. 范围: 从 0 到 1                                             │
│ 3. 分位数: CDF = 0.25 对应 Q1, CDF = 0.5 对应中位数             │
│ 4. 百分位解读: CDF(x) = p 意味着 x 是第 (p×100) 百分位         │
└────────────────────────────────────────────────────────────────┘
```

### 3.3 为什么 CDF 对 Baseline 设定很重要？

CDF 图提供了**直观的数据分布视图**，让我们能够：

1. **看到阈值影响的范围**: CDF(阈值) 直接告诉我们多少比例的数据在阈值以下
2. **识别拐点**: CDF 曲线的斜率变化反映数据密度的变化
3. **比较不同阈值**: 一目了然地看到不同阈值对应的通过率

## 4. Baseline 选择的统计学依据

### 4.1 环境泛化性 Baseline: ≤ 0.38

**为什么选择 0.38？**

```
统计位置分析:
┌─────────────────────────────────────────────────────────────────┐
│  清洗后数据: μ = 0.448, σ = 0.068                               │
│                                                                 │
│  0.38 = μ - 0.99σ ≈ μ - 1σ                                     │
│                                                                 │
│  在正态分布中:                                                  │
│  • μ - 1σ 对应约第 16 百分位                                    │
│  • 意味着选择分布中最优的 ~16% 数据                             │
└─────────────────────────────────────────────────────────────────┘
```

**CDF 图上的位置:**
```
CDF
1.0 ┤                                    ●────
    │                                 ●
0.8 ┤                              ●
    │                           ●
0.6 ┤                        ●
    │                     ●
0.4 ┤                  ●
    │               ●
0.2 ┤         ●───●           ← 拐点区域！
    │      ●  ↑
0.1 ┤   ●     │
    │  ●      0.38 (Baseline)
0.0 ┼●────────│─────────────────────────────→ Env Gen
    0.2     0.38    0.45    0.55    0.65

解读:
- 0.38 处 CDF ≈ 0.16，意味着 16% 数据 ≤ 0.38
- 这个位置恰好在 CDF 曲线从平缓变陡峭的拐点附近
- 选择此点可以有效区分"优质"和"普通"数据
```

**σ 的意义:**
```
在正态分布中，μ - 1σ 的统计意义:

         μ-2σ    μ-1σ     μ      μ+1σ    μ+2σ
           │       │       │       │       │
    ───────┼───────┼───────┼───────┼───────┼───────
           │       │◄──────────────►│       │
           │       │     68.3%      │       │
           │◄──────────────────────────────►│
           │            95.4%              │
           
   μ - 1σ 将数据分为:
   • 左侧 ~16%: 显著优于平均 (我们要的！)
   • 右侧 ~84%: 平均或以下
```

### 4.2 夹爪熵 Baseline: ≥ 6.49

**为什么选择 6.49？**

```
统计位置分析:
┌─────────────────────────────────────────────────────────────────┐
│  清洗后数据: μ = 6.18, σ = 0.61                                 │
│                                                                 │
│  6.49 = μ + 0.51σ ≈ μ + 0.5σ                                   │
│                                                                 │
│  在正态分布中:                                                  │
│  • μ + 0.5σ 对应约第 69 百分位                                  │
│  • 意味着选择分布中最优的 ~31% 数据                             │
└─────────────────────────────────────────────────────────────────┘
```

**CDF 图上的位置:**
```
CDF
1.0 ┤                              ●────────
    │                           ●
0.8 ┤                        ●
    │                     ●───●    ← 拐点区域！
0.6 ┤                  ●  ↑
    │               ●     │
0.4 ┤            ●        6.49 (Baseline)
    │         ●
0.2 ┤      ●
    │   ●
0.0 ┼●─────────────────────│────────────────→ Gripper Entropy
    4.0   5.0    5.5    6.0  6.49  6.8   7.2

解读:
- 6.49 处 CDF ≈ 0.61，意味着 39% 数据 ≥ 6.49
- 这个位置在 CDF 曲线斜率开始减小的拐点区域
- 选择此点可以保留动作多样性较高的数据
```

**σ 的意义:**
```
在正态分布中，μ + 0.5σ 的统计意义:

         μ-1σ     μ      μ+0.5σ   μ+1σ    μ+2σ
           │       │       │       │       │
    ───────┼───────┼───────┼───────┼───────┼───────
                   │◄─────►│       │
                   │ 19.1% │       │
                   │◄─────────────►│
                   │     34.1%    │
           
   μ + 0.5σ 将数据分为:
   • 右侧 ~31%: 显著优于平均 (我们要的！)
   • 左侧 ~69%: 平均或以下
```

### 4.3 为什么这两个阈值是"拐点"？

**拐点的统计学意义:**

CDF 曲线的"拐点"是斜率变化显著的区域，对应于**概率密度函数 (PDF) 的峰值区域边界**。

```
PDF 与 CDF 的关系:

PDF (概率密度):                    CDF (累积分布):
                                   
     │    ╱╲                            │        ●────
     │   ╱  ╲                           │      ●
     │  ╱    ╲                          │    ●
     │ ╱      ╲                         │  ●      ← 拐点 = 斜率最大处
     │╱        ╲                        │●
     └──────────→                       └──────────→
         ↑                                  ↑
       峰值                               拐点
       
PDF 峰值处 = CDF 斜率最大处 (拐点)
```

**在我们的数据中:**

- **0.38 (环境泛化性)**: 位于 PDF 左侧峰值下降的起点，是"优质数据"的边界
- **6.49 (夹爪熵)**: 位于 PDF 右侧峰值的位置，是"高多样性"的起点

选择拐点作为阈值的好处：
1. **最大化区分度**: 拐点是数据密度变化最快的位置
2. **统计学意义明确**: 对应于 σ 的整数或半整数倍
3. **实际可操作**: 不会过于严格也不会过于宽松

## 5. 综合分析：为什么 0.38 和 6.49 是最优选择

### 5.1 统计学维度

| 指标 | Baseline | σ 位置 | 百分位 | 通过率 |
|------|----------|--------|--------|--------|
| 环境泛化性 | ≤ 0.38 | μ - 1σ | 16th | 16.2% |
| 夹爪熵 | ≥ 6.49 | μ + 0.5σ | 69th | 38.6% |

### 5.2 实际意义

```
选择这两个阈值的实际意义:

环境泛化性 ≤ 0.38:
┌─────────────────────────────────────────────────────────────────┐
│ • 环境泛化性衡量的是采集第一帧与参考的差异                        │
│ • 越小 = 环境越一致、采集起始状态越标准                          │
│ • 0.38 = 前 16% 最稳定的采集环境                                │
│ • 保证了训练数据的环境一致性                                    │
└─────────────────────────────────────────────────────────────────┘

夹爪熵 ≥ 6.49:
┌─────────────────────────────────────────────────────────────────┐
│ • 夹爪熵衡量的是夹爪开合动作的多样性                              │
│ • 越大 = 动作越丰富、轨迹越多样                                  │
│ • 6.49 = 前 39% 最多样的动作轨迹                                │
│ • 保证了训练数据的动作多样性                                    │
└─────────────────────────────────────────────────────────────────┘
```

### 5.3 平衡考量

```
为什么不选择更严格或更宽松的阈值？

更严格 (如 μ - 2σ 和 μ + 1σ):
  ✗ 通过率太低 (< 5%)
  ✗ 可能丢失大量可用数据
  ✗ 采集成本过高

更宽松 (如 μ - 0.5σ 和 μ):
  ✗ 质量控制不够
  ✗ 允许过多低质量数据进入训练集
  ✗ 可能影响模型性能

当前选择 (μ - 1σ 和 μ + 0.5σ):
  ✓ 通过率合理 (~7% 同时满足两条件)
  ✓ 有效过滤低质量数据
  ✓ 基于 CDF 拐点，区分度最大
  ✓ 统计学意义明确
```
